# 🛡️ Улучшение системы дедупликации новостей

## 📋 Проблема
Новости публиковались повторно, если они были написаны по-разному, но об одном и том же событии. Например:
- "Макрон созвал срочное заседание Совета Безопасности ООН"
- "Срочный созыв СБ ООН по инициативе президента Франции"

Обе новости об одном событии, но система не распознавала их как дубликаты.

## ✅ Что было сделано

### 1. Улучшение `global_dedup.py`

#### Новая функция нормализации текста
- **Расширенные списки** для замены: имена, страны, города, организации, валюты
- **Специальные правила** для составных терминов (СБ ООН → UNSC, США → USA)
- **Нормализация действий** (созвал, инициировал → CALLED)
- **Нормализация событий** (заседание, совещание → MEETING)

#### Новые функции проверки
```python
# Семантический хеш (ключевые сущности без действий)
get_semantic_hash(title, category)

# Проверка схожести текстов (fuzzy matching)
texts_are_similar(text1, text2, threshold=0.80)

# Извлечение ключевых сущностей
extract_key_entities(title)

# Сравнение по сущностям
are_duplicates_by_entities(title1, title2, min_common_entities=2)

# Многоуровневая проверка
check_duplicate_multi_layer(title, body, category, hours=48)
```

#### Уровни проверки дубликатов
1. **Content Hash** — строгая нормализация заголовка + тела
2. **Semantic Hash** — только ключевые сущности
3. **Fuzzy Matching** — проверка схожести текста (75%+)
4. **Entity Matching** — сравнение по ключевым сущностям

## 🔧 Как использовать

### В ботах
```python
from global_dedup import check_duplicate_multi_layer, mark_global_posted

# Проверка перед публикацией
is_dup, reason = await check_duplicate_multi_layer(
    title, summary, category, hours=48
)
if is_dup:
    logger.info(f"🔄 Дубликат ({reason}): {title[:50]}")
    return

# После успешной публикации
content_hash = get_content_dedup_hash(title, summary, category)
await mark_global_posted(content_hash, channel_id, parser_name, title, category)
```

## 🐛 Исправления

### Ошибка "get_content_dedup_hash() takes 2 positional arguments but 3 were given"

**Проблема:** В каждом боте была своя локальная функция `get_content_dedup_hash(title, body)` без параметра категории, но новый код вызывал её с тремя аргументами.

**Решение:** Обновлены локальные функции в `movie.py`, `economika.py`, `politika.py` для делегирования глобальной функции из `global_dedup.py`:

```python
def get_content_dedup_hash(title: str, body: str, category: str = "") -> str:
    from global_dedup import get_content_dedup_hash as global_hash
    return global_hash(title, body, category)
```

### Блокировка по английским словам

**Проблема:** LLM в рерайте использовал английские названия городов (Dubai, Tehran) и отраслевые термины (airports, flights), что блокировалось фильтром.

**Решение:** Расширен список `allowed_english` в `economika.py` и `politika.py`:
- Географические названия: `dubai`, `dxb`, `dwc`, `tehran`, `israel`, и т.д.
- Отраслевые термины: `airports`, `flights`, `oil`, `gas`, `military`

Также улучшен промпт для LLM с явным указанием что МОЖНО использовать эти слова.
